# Multi-Agent Interview Coach

AI-тренажёр технических собеседований, построенный на мультиагентной архитектуре. Система симулирует настоящее интервью: задаёт вопросы, адаптируется под уровень кандидата, проверяет факты и в конце выдаёт подробный фидбек.

## Зачем это нужно

Подготовка к техническим интервью — это стресс. Хочется потренироваться, но просить друзей проводить mock-интервью неудобно, а платные сервисы дорогие. Этот проект решает проблему: запускаешь локально, тренируешься сколько хочешь, получаешь честную обратную связь.

## Как устроена система

Вместо одного большого промпта я разбил логику на 7 специализированных агентов. Каждый делает что-то одно, но делает это хорошо:

```
Кандидат отвечает
       ↓
[Answer Analyzer] — понимает, что ответил кандидат, насколько полно и по теме
       ↓
[Fact Checker] — если в ответе есть подозрительные утверждения, гуглит их
       ↓
[Evaluator] — обновляет накопительную оценку навыков
       ↓
[Router] — решает, что делать дальше: углубиться, сменить тему, завершить
       ↓
[Interviewer] — формулирует следующий вопрос или реплику
```

Плюс два агента работают в начале и конце:
- **Topic Planner** — в начале составляет план интервью под конкретную позицию
- **Hiring Manager** — в конце собирает всё вместе и пишет итоговый отчёт

### Почему агенты, а не один промпт?

1. **Проще отлаживать** — если Answer Analyzer криво оценивает ответы, фиксишь только его
2. **Легче расширять** — хочешь добавить проверку кода? Добавь Code Evaluator агента
3. **Предсказуемость** — каждый агент возвращает строго типизированный JSON через Pydantic

### Технологии

- **LangGraph** — оркестрация агентов как граф состояний с условными переходами
- **LangChain** — обёртка над LLM с Structured Output
- **OpenAI GPT-4o-mini** — основная модель (быстрая и недорогая)
- **DuckDuckGo** — веб-поиск для проверки фактов
- **Gradio** — веб-интерфейс

## Быстрый старт

### 1. Клонируй и установи зависимости

```bash
python -m venv venv
venv\Scripts\activate  # Windows
# source venv/bin/activate  # Linux/Mac

pip install -r requirements.txt
```

### 2. Настрой API ключи

```bash
copy .env.example .env
```

Открой `.env` и вставь свой OpenAI API ключ:

```
OPENAI_API_KEY=sk-...
```

Где взять: https://platform.openai.com/api-keys

### 3. Запусти

```bash
python run.py
```

Открой http://127.0.0.1:7860 в браузере.

## Примеры работы

### Начало интервью

Заполняешь форму: имя, позиция (например, "Python Backend Developer"), грейд (Junior/Middle/Senior), краткий опыт. Нажимаешь "Начать интервью".

Система составляет план тем и задаёт первый вопрос:

> Привет! Меня зовут AI Interviewer, буду проводить техническое интервью на позицию Python Backend Developer. Расскажи, пожалуйста, что ты знаешь о GIL в Python и как он влияет на многопоточность?

### Адаптивная сложность

Если отвечаешь хорошо — вопросы усложняются:
> Отлично! А как бы ты решил проблему CPU-bound задач в Python, учитывая GIL?

Если отвечаешь слабо — система даёт подсказку и спрашивает проще:
> Давай попроще. Что такое потоки и процессы? В чём разница?

### Проверка фактов

Если скажешь что-то сомнительное:
> "В Python 4.0 добавили нативную компиляцию в машинный код"

Fact Checker погуглит и мягко поправит:
> Интересно! Правда, я не нашёл подтверждения, что Python 4.0 уже вышел. Возможно, ты имел в виду что-то другое?

### Финальный отчёт

После команды "стоп интервью" Hiring Manager формирует отчёт:

- **Рекомендация**: Hire / No Hire / Maybe
- **Сильные стороны**: что показал хорошо
- **Зоны роста**: что подтянуть
- **Оценка по темам**: Python basics — 8/10, Async — 6/10, и т.д.

## Структура проекта

```
├── run.py                 # Точка входа
├── src/
│   ├── main.py            # Gradio UI
│   ├── config.py          # Настройки из .env
│   │
│   ├── agents/            # Агенты
│   │   ├── topic_planner.py
│   │   ├── interviewer.py
│   │   ├── answer_analyzer.py
│   │   ├── fact_checker.py
│   │   ├── evaluator.py
│   │   ├── question_handler.py
│   │   └── hiring_manager.py
│   │
│   ├── graph/
│   │   └── interview_graph.py   # LangGraph StateGraph
│   │
│   ├── models/
│   │   ├── state.py             # Состояние интервью
│   │   ├── schemas.py           # Pydantic модели
│   │   └── output_schemas.py    # Схемы для Structured Output
│   │
│   ├── prompts/
│   │   └── templates.py         # Промпты агентов
│   │
│   └── tools/
│       └── web_search.py        # DuckDuckGo поиск
```

## Конфигурация

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `OPENAI_API_KEY` | Да | API ключ OpenAI |
| `OPENAI_MODEL` | Нет | Модель (по умолчанию gpt-4o-mini) |
